# 小智智能语音设备 - 语音转文本与文本转语音功能用法总结

## 项目概述

小智智能语音设备是一个基于ESP32-S3平台的智能语音助手，采用本地唤醒+云端处理的混合架构。项目主要功能包括语音唤醒、语音识别（STT）、文本转语音（TTS）和云端交互。

**核心技术栈**：
- **硬件平台**：ESP32-S3
- **操作系统**：FreeRTOS
- **音频处理**：ESP-SR（语音识别）、ESP-TTS（文本转语音）
- **图形界面**：LVGL
- **通信协议**：WebSocket

**架构特点**：
- 本地唤醒词检测（"你好小智"）
- 云端语音识别和语音合成
- 事件驱动的模块化设计
- 7状态有限状态机管理

## 语音转文本（STT）功能

### 实现原理

设备采用**云端STT服务**，本地只负责唤醒词检测和音频采集/上传：

1. **本地处理**：
   - 实时音频采集（16kHz, 16位）
   - 唤醒词检测（ESP-SR的`esp_afe_sr_iface_t`接口）
   - 语音活动检测（VAD）
   - Opus音频编码

2. **云端处理**：
   - 音频数据通过WebSocket上传到服务器
   - 服务器进行语音识别（STT）
   - 返回文本识别结果

### 关键文件

| 文件路径 | 功能描述 |
|---------|----------|
| [main/audio/audio_sr.c](main/audio/audio_sr.c) | 语音识别核心，唤醒词检测和VAD |
| [main/audio/audio_processor.c](main/audio/audio_processor.c) | 音频处理器，协调录音和播放 |
| [main/audio/audio_encoder.c](main/audio/audio_encoder.c) | Opus音频编码，用于上传 |
| [main/protocol/protocol.c](main/protocol/protocol.c) | STT结果处理和网络通信 |

### STT事件流程

```c
// 1. 唤醒词检测（本地）
if (result->wakeup_state == WAKENET_DETECTED) {
    esp_event_post_to(sr->event_loop, AUDIO_SR_EVENT,
                      AUDIO_PROCESSOR_EVENT_WAKEUP, NULL, 0, 0);
}

// 2. 开始录音并上传（应用层）
case AUDIO_PROCESSOR_EVENT_SPEECH:
    if (app->state == APP_STATE_WAKEUP) {
        application_set_state(app, APP_STATE_LISTENING);
        protocol_send_start_listening(app->protocol, PROTOCOL_LISTEN_TYPE_AUTO);
    }
    break;

// 3. 接收STT结果（云端返回）
case PROTOCOL_EVENT_STT: // event_data为char*
    ESP_LOGI(TAG, "STT: %s", (char *)event_data);
    ui_update_text((char *)event_data); // 更新UI显示
    break;
```

### 音频配置参数

- **采样率**：16000 Hz
- **位深度**：16位
- **音频格式**：PCM → Opus编码
- **帧时长**：60ms
- **VAD阈值**：通过ESP-SR配置

### 状态转移

STT功能涉及以下状态转移：
- `APP_STATE_IDLE` → `APP_STATE_WAKEUP`（唤醒词检测）
- `APP_STATE_WAKEUP` → `APP_STATE_LISTENING`（检测到语音）
- `APP_STATE_LISTENING` → `APP_STATE_WAKEUP`（静音检测）

## 文本转语音（TTS）功能

### 实现原理

设备采用**云端TTS服务**，本地负责音频接收和解码播放：

1. **云端处理**：
   - 服务器接收文本并进行TTS合成
   - 生成Opus编码的音频流

2. **本地处理**：
   - 通过WebSocket接收音频数据
   - Opus解码为PCM格式
   - 通过扬声器播放

### 关键文件

| 文件路径 | 功能描述 |
|---------|----------|
| [main/protocol/protocol.c](main/protocol/protocol.c) | TTS事件处理和音频数据接收 |
| [main/audio/audio_decoder.c](main/audio/audio_decoder.c) | Opus音频解码，用于播放 |
| [main/audio/audio_processor.c](main/audio/audio_processor.c) | 音频写入和播放控制 |
| [main/application.c](main/application.c) | TTS事件状态处理 |

### TTS事件类型

设备支持三种TTS事件：

| 事件类型 | 事件数据 | 说明 |
|---------|----------|------|
| `PROTOCOL_EVENT_TTS_START` | NULL | TTS开始，设备进入讲话状态 |
| `PROTOCOL_EVENT_TTS_SENTENCE_START` | char* (文本) | 句子开始，包含要合成的文本 |
| `PROTOCOL_EVENT_TTS_STOP` | NULL | TTS结束，设备返回唤醒状态 |
| `PROTOCOL_EVENT_AUDIO` | binary_data_t* (音频数据) | 音频数据包，包含Opus编码的音频 |

### TTS处理流程

```c
// 1. TTS开始事件
case PROTOCOL_EVENT_TTS_START:
    if (app->state == APP_STATE_WAKEUP) {
        audio_processor_set_vad_state(app->processor, false); // 关闭VAD
        application_set_state(app, APP_STATE_SPEAKING);      // 进入讲话状态
    }
    break;

// 2. TTS句子开始（显示文本）
case PROTOCOL_EVENT_TTS_SENTENCE_START:
    ESP_LOGI(TAG, "TTS: %s", (char *)event_data);
    ui_update_text((char *)event_data); // 在UI上显示文本
    break;

// 3. 接收音频数据并播放
case PROTOCOL_EVENT_AUDIO:
    if (app->state == APP_STATE_SPEAKING) {
        binary_data_t *data = (binary_data_t *)event_data;
        audio_processor_write(app->processor, data->ptr, data->size); // 播放音频
    }
    break;

// 4. TTS结束事件
case PROTOCOL_EVENT_TTS_STOP:
    if (app->state == APP_STATE_SPEAKING) {
        application_set_state(app, APP_STATE_WAKEUP);        // 返回唤醒状态
        audio_processor_set_vad_state(app->processor, true); // 重新开启VAD
    }
    break;
```

### 音频播放API

```c
/**
 * @brief 播放Opus编码的声音
 * @param audio_processor 音频处理器实例
 * @param buffer Opus编码的音频数据
 * @param size 数据大小
 */
void audio_processor_write(audio_processor_t *audio_processor, void *buffer, size_t size);
```

### 状态转移

TTS功能涉及以下状态转移：
- `APP_STATE_WAKEUP` → `APP_STATE_SPEAKING`（收到TTS_START）
- `APP_STATE_SPEAKING` → `APP_STATE_WAKEUP`（收到TTS_STOP）
- `APP_STATE_SPEAKING` → `APP_STATE_WAKEUP`（打断唤醒）

## 使用流程示例

### 完整语音交互流程

1. **设备启动** → 初始化硬件 → 检查激活 → 进入空闲状态
2. **语音唤醒** → 用户说"你好小智" → 本地唤醒词检测 → 连接服务器
3. **语音识别** → 检测到语音 → 开始录音 → Opus编码 → 上传服务器
4. **云端处理** → STT识别 → LLM分析 → TTS合成
5. **语音播放** → 接收TTS音频流 → Opus解码 → PCM播放

### 代码示例：STT和TTS集成

```c
// 事件处理回调函数
static void app_event_handler(void *arg, esp_event_base_t base, int32_t event_id, void *event_data) {
    application_t *app = (application_t *)arg;

    switch (event_id) {
        // STT相关事件
        case PROTOCOL_EVENT_STT:
            ESP_LOGI(TAG, "识别结果: %s", (char *)event_data);
            ui_update_text((char *)event_data);
            break;

        // TTS相关事件
        case PROTOCOL_EVENT_TTS_START:
            ESP_LOGI(TAG, "TTS开始");
            application_set_state(app, APP_STATE_SPEAKING);
            break;

        case PROTOCOL_EVENT_TTS_SENTENCE_START:
            ESP_LOGI(TAG, "TTS文本: %s", (char *)event_data);
            ui_update_text((char *)event_data);
            break;

        case PROTOCOL_EVENT_AUDIO:
            if (app->state == APP_STATE_SPEAKING) {
                binary_data_t *data = (binary_data_t *)event_data;
                audio_processor_write(app->processor, data->ptr, data->size);
            }
            break;

        case PROTOCOL_EVENT_TTS_STOP:
            ESP_LOGI(TAG, "TTS结束");
            application_set_state(app, APP_STATE_WAKEUP);
            break;
    }
}
```

## 性能指标

### 响应时间要求
- **唤醒检测**：< 200ms
- **网络连接**：1-3秒
- **STT处理**：云端处理时间
- **TTS播放**：实时播放，无缓冲延迟

### 超时控制
- **唤醒超时**：5秒无语音输入 → 断开连接
- **录音超时**：最长60秒录音限制
- **网络超时**：连接建立10秒超时

## 故障排除

### 常见问题及解决方法

1. **STT无响应**
   - 检查网络连接状态
   - 验证WebSocket连接是否建立
   - 检查音频编码是否正确

2. **TTS无声**
   - 确认设备处于`APP_STATE_SPEAKING`状态
   - 检查音频解码器是否正常初始化
   - 验证扬声器硬件连接

3. **唤醒词不灵敏**
   - 调整麦克风增益
   - 检查环境噪音水平
   - 更新ESP-SR唤醒词模型

4. **音频质量差**
   - 检查采样率配置（必须为16000Hz）
   - 验证Opus编码参数
   - 检查硬件Codec配置

## 总结

小智智能语音设备的STT和TTS功能采用了**云端协同架构**，平衡了设备成本与功能复杂度：

1. **STT功能**：本地唤醒+云端识别，保证了唤醒实时性和识别准确性
2. **TTS功能**：云端合成+本地播放，提供了高质量的语音输出
3. **架构优势**：模块化设计、事件驱动、状态机管理确保了系统稳定性
4. **扩展性**：支持本地TTS扩展、自定义唤醒词和音频格式扩展

这种架构既利用了云端强大的AI能力，又通过本地处理保证了实时性和隐私性，是嵌入式智能语音设备的典型设计方案。
