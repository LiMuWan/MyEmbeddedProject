# 小智智能语音设备项目架构与业务逻辑分析

## 项目概述

**项目名称**：小智智能语音交互设备
**项目类型**：基于ESP32-S3的嵌入式智能语音设备
**技术栈**：ESP-IDF 5.4.2 + FreeRTOS + LVGL + ESP-SR
**应用场景**：智能语音助手，支持语音唤醒、语音识别、语音合成、云端交互

## 核心模块分析

### 1. 应用层 (Application Layer)

#### 状态机设计
应用层采用**7状态有限状态机**管理设备行为：

```c
typedef enum {
    APP_STATE_STARTING,     // 启动中
    APP_STATE_ACTIVATING,   // 激活中（显示激活码）
    APP_STATE_IDLE,         // 空闲（等待唤醒）
    APP_STATE_CONNECTING,   // 连接服务器中
    APP_STATE_WAKEUP,       // 已唤醒（等待语音输入）
    APP_STATE_LISTENING,    // 正在监听（录音中）
    APP_STATE_SPEAKING,     // 正在讲话（播放TTS）
} app_state_t;
```

#### 状态转移图
```
┌──────────┐    激活完成    ┌──────────┐    唤醒词    ┌──────────┐
│ 启动中   │ ────────────> │  空闲    │ ───────────> │ 连接中   │
└──────────┘               └──────────┘              └──────────┘
      │                          │                         │
      │ 需要激活                 │ 连接成功                 │ 连接成功
      ↓                          ↓                         ↓
┌──────────┐               ┌──────────┐               ┌──────────┐
│ 激活中   │               │  唤醒    │ <─────────── │  唤醒    │
└──────────┘               └──────────┘    TTS结束    └──────────┘
      │                          │                         │
      │ 显示激活码               │ 检测到语音               │ 播放音频
      ↓                          ↓                         ↓
                               ┌──────────┐               ┌──────────┐
                               │ 监听中   │               │ 讲话中   │
                               └──────────┘               └──────────┘
                                     │                         │
                                     │ 静音检测                 │ 打断唤醒
                                     ↓                         ↓
```

### 2. 音频处理模块 (Audio Processing)

#### 功能职责
- **音频采集**：通过Codec采集麦克风输入
- **语音唤醒**：使用ESP-SR识别唤醒词"你好小智"
- **语音活动检测(VAD)**：检测语音开始和结束
- **音频编码**：将PCM音频编码为Opus格式
- **音频解码**：将Opus音频解码为PCM播放

#### 事件系统
```c
typedef enum {
    AUDIO_PROCESSOR_EVENT_WAKEUP,  // 唤醒词检测
    AUDIO_PROCESSOR_EVENT_SPEECH,  // 语音开始
    AUDIO_PROCESSOR_EVENT_SILENCE, // 静音检测
} audio_processor_event_t;
```

### 3. 通信协议模块 (Protocol)

#### WebSocket协议设计
- **连接地址**：从OTA服务器获取WebSocket URL和Token
- **消息格式**：JSON格式，支持双向通信
- **心跳机制**：保持长连接

#### 协议事件
```c
typedef enum {
    PROTOCOL_EVENT_CONNECTED,           // 连接成功
    PROTOCOL_EVENT_DISCONNECTED,        // 连接断开
    PROTOCOL_EVENT_HELLO,               // 服务器问候
    PROTOCOL_EVENT_STT,                 // 语音识别结果
    PROTOCOL_EVENT_LLM,                 // LLM情感分析
    PROTOCOL_EVENT_TTS_START,           // TTS开始
    PROTOCOL_EVENT_TTS_SENTENCE_START,  // TTS句子开始
    PROTOCOL_EVENT_TTS_STOP,            // TTS结束
    PROTOCOL_EVENT_AUDIO,               // 音频数据
} protocol_event_t;
```

### 4. OTA升级模块

#### 激活机制
1. 设备首次启动时从`https://api.tenclass.net/xiaozhi/ota/`获取激活信息
2. 如果未激活，显示激活码二维码
3. 用户扫码激活后，获取WebSocket连接信息

#### 数据结构
```c
typedef struct {
    char* activation_code;     // 激活码
    char* websocket_url;       // WebSocket服务器地址
    char* websocket_token;     // 连接令牌
} ota_t;
```

## 业务逻辑流程

### 1. 设备启动流程
```
1. 硬件初始化 (BSP)
   ├── LCD初始化 → 显示启动界面
   ├── LED/按钮初始化
   ├── NVS存储初始化
   ├── WiFi初始化 → 显示配网二维码(如需)
   └── 音频Codec初始化

2. OTA激活检查
   ├── 查询激活服务器
   ├── 如果未激活 → 显示激活码 → 等待激活
   └── 如果已激活 → 获取WebSocket连接信息

3. 模块初始化
   ├── 创建协议模块 (WebSocket)
   ├── 创建音频处理器
   └── 注册事件回调

4. 进入空闲状态
   └── 等待语音唤醒
```

### 2. 语音交互流程
```
1. 唤醒阶段
   用户说"你好小智" → 音频处理器检测唤醒词 → 触发WAKEUP事件

2. 连接阶段
   应用层状态: IDLE → CONNECTING
   建立WebSocket连接 → 发送HELLO消息

3. 监听阶段
   连接成功 → 状态: CONNECTING → WAKEUP
   开启VAD检测 → 等待用户说话
   检测到语音 → 状态: WAKEUP → LISTENING
   开始录音并上传音频数据

4. 处理阶段
   静音检测 → 停止录音 → 状态: LISTENING → WAKEUP
   服务器返回:
     - STT结果 → 显示文本
     - LLM情感 → 更新情感状态
     - TTS开始 → 状态: WAKEUP → SPEAKING
     - 音频数据 → 播放TTS音频

5. 结束阶段
   TTS播放完成 → 状态: SPEAKING → WAKEUP
   超时(5秒无交互) → 断开连接 → 状态: WAKEUP → IDLE
```

### 3. 异常处理流程
```
1. 网络断开
   PROTOCOL_EVENT_DISCONNECTED → 状态: 当前状态 → IDLE
   关闭VAD检测 → 等待重新唤醒

2. 语音打断
   在SPEAKING状态检测到唤醒词 → 中止当前TTS
   发送abort_speaking消息 → 重新进入WAKEUP状态

3. 硬件故障
   BSP状态检查失败 → 设备启动失败
   记录错误日志 → 进入安全模式
```

## 数据流分析

### 音频数据流
```
麦克风输入 → Codec采集(PCM) → 音频处理器 → Opus编码 → WebSocket上传
                                 ↓
                         唤醒词检测(VAD)
                                 ↓
                           事件触发
```

### 云端数据流
```
设备端 → WebSocket → 云端服务器
   ↑                      ↓
   ├── 音频数据 ─────> STT服务 → 文本
   ├── 文本 ────────> LLM服务 → 情感+回复
   └── 回复文本 ────> TTS服务 → 音频数据
```

### 控制流
```
硬件事件 → BSP层 → 应用层状态机 → 协议层 → 云端
   ↑                                          ↓
   └────────────────── 响应处理 ───────────────┘
```

## 关键技术特性

### 1. 实时性设计
- **FreeRTOS任务调度**：音频处理、网络通信、UI更新分离
- **SPI RAM使用**：大容量音频缓冲区
- **事件驱动架构**：低延迟响应

### 2. 电源管理
- LCD背光控制
- WiFi节能模式
- 空闲状态低功耗

### 3. 网络可靠性
- WebSocket自动重连
- 心跳保活机制
- 离线缓存支持

### 4. 用户体验
- 语音唤醒响应时间 < 200ms
- TTS播放流畅
- 实时状态反馈
